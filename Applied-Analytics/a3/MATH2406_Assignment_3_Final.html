Why Not Watch? Recommendation Engine A/B Test Analysis

Critical Assessment of Algorithm Effectiveness and Experimental Design

Michael Teixeira S4133975

Last updated: 15 June, 2025

Executive Summary

Key Finding: The new recommendation algorithm shows a 10.9% increase in daily viewing hours, but severe methodological flaws make this result unreliable for business decisions.

Critical Issues: - Extreme group imbalance (88% control vs 12% treatment) - Temporal confounding (before/after design, not true randomization) - Demographic bias between groups

Recommendation: Do not roll out algorithm company-wide. Conduct proper randomized trial first.

Business Context: WNW seeks to evaluate new recommendation engine. Algorithm implemented July 18th at 00:01. Hours watched per day drives advertising revenue.

Problem Statement & Data Overview

Research Question: Does the new recommendation engine significantly increase daily viewing hours?

Hypotheses: \(H_0: \mu_B - \mu_A = 0\) vs \(H_A: \mu_B - \mu_A \neq 0\)

# Primary outcome analysis
outcome_summary <- streaming %>%
  group_by(group) %>%
  summarise(
    n = n(),
    Mean_Hours = round(mean(hours_watched), 3),
    SD_Hours = round(sd(hours_watched), 3),
    .groups = 'drop'
  )

# Effect size calculation
group_a_mean <- outcome_summary$Mean_Hours[1]
group_b_mean <- outcome_summary$Mean_Hours[2]
raw_effect <- group_b_mean - group_a_mean
percent_increase <- round((raw_effect / group_a_mean) * 100, 1)

# Primary t-test
primary_test <- t.test(hours_watched ~ group, data = streaming)

# Calculate Cohen's d
group_a_sd <- sd(streaming$hours_watched[streaming$group == "A"])
group_b_sd <- sd(streaming$hours_watched[streaming$group == "B"])
pooled_sd <- sqrt(((879 * group_a_sd^2) + (119 * group_b_sd^2)) / (880 + 120 - 2))
cohens_d <- (group_b_mean - group_a_mean) / pooled_sd

Discussion & Final Recommendations

Major Findings

  1. Statistical Significance: 10.9% increase (p < 0.001)
  2. Medium Effect Size: Cohens d = 0.359
  3. Fatal Bias Issues: Multiple confounding sources undermine validity

Critical Limitations

  • Not a true A/B test: Before/after comparison disguised as randomization
  • Severe group imbalance: 7.33:1 ratio reduces power to ~30%
  • Demographic bias: Age and gender differences between groups
  • Temporal confounding: Algorithm change coincides with date split

Business Decision Framework

IMMEDIATE RECOMMENDATION: DO NOT ROLL OUT

Risk Assessment: 70% probability the observed effect is due to bias rather than algorithm effectiveness

Action Plan: 1. Phase 1 (2 weeks): Design proper RCT with 1:1 randomization 2. Phase 2 (6 weeks): Execute balanced trial with 267 users per group
3. Phase 3 (1 week): Make confident go/no-go decision

Investment Logic: Spend $50K on proper study design to protect $2-5M implementation decision

Bottom Line

The 10.9% improvement cannot be confidently attributed to the algorithm due to severe methodological flaws. A proper randomized trial is essential before company-wide implementation.
